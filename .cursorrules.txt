## Development Guidelines
1. **Component Structure**: Create reusable components for sections
2. **State Management**: Use React hooks for local state, Context for global state
3. **Performance**: Lazy load images, optimize bundle size
4. **SEO**: Proper meta tags, structured data, sitemap
5. **Analytics**: Google Analytics integration ready
6. **Forms**: Use libraries like React Hook Form for validation

## Code Style
- Use functional components with hooks
- TypeScript for type safety
- ESLint + Prettier for code formatting
- Consistent naming conventions (camelCase for variables, PascalCase for components)
- Meaningful component and variable names
- Comprehensive comments for complex logic

## Responsive Design
- Mobile: 320px - 768px
- Tablet: 768px - 1024px
- Desktop: 1024px+
- Use Tailwind's responsive prefixes (sm:, md:, lg:, xl:)

## Animation Guidelines
- Smooth page transitions
- Hover effects on interactive elements
- Scroll-triggered animations (fade-in, slide-up)
- Loading animations
- Subtle micro-interactions
- Performance-optimized animations (60fps)

## Content Guidelines
- Use professional, technical language appropriate for data science field
- Quantify achievements with numbers and percentages
- Keep descriptions concise but informative
- Maintain consistent tone throughout
- Ensure all technical terms are accurate
